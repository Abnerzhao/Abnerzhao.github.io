  • RAID阵列卡系列命令备忘

    Oct 11, 2015

    阵列卡的常见类型

    查看阵列卡的类型

    lspci | grep -i raid

    一般有lsi和hp的两种，对应的管理命令为 MegaCli 和 hpssacli

    LSI官方网站和驱动下载 avagotech

    常用命令

    LSI的MegaSAS RAID卡系列命令

    显示所有逻辑磁盘组信息

    MegaCli -LDInfo -Lall -aALL
MegaCli -LDInfo -Lall -aALL | grep State
// 过滤State， 正常情况为："State : Optimal"，否则为异常，例如 "State : Degraded"

    查看所有物理磁盘信息

    MegaCli -PDList -aAll -NoLog
MegaCli -PDList -aAll -NoLog | grep   "Firmware state"
//如果 "Firmware state" 显示为非 Online，则表示该磁盘有问题。

    显示 RAID 卡型号，RAID 设置及物理磁盘的相关信息

    MegaCli -cfgdsply -aALL -NoLog

    查看磁盘总数和磁盘类型

    MegaCli -cfgdsply -aALL -NoLog | grep "Media Type:"

    查看磁盘的rebuild 进度

    # MegaCli -PDRbld -ShowProg -PhysDrv [32:3] -a0
Rebuild Progress on Device at Enclosure 32, Slot 3 Completed 95% in 50 Minutes.

    显示 RAID 卡信息

    MegaCli -AdpAllInfo -aALL -NoLog

    查看 RAID 卡日志

    MegaCli -FwTermLog -Dsply -aALL -NoLog

    查看电池信息

    MegaCli -AdpBbuCmd -aAll -NoLog

    显示适配器个数

    MegaCli -adpCount -NoLog

    显示适配器时间

    MegaCli -AdpGetTime –aALL -NoLog

    常见设置

    MegaCli -LDSetProp WB -LALL -aALL
MegaCli -LDSetProp ADRA -LALL-aALL
MegaCli -LDSetProp disDskCache -LALL-aALL
MegaCli -LDSetProp CachedBadBBU -Lall -aALL

    HP Smart Array CLI 命令

    显示raid卡信息

    hpssacli ctrl all show  config
hpssacli ctrl all show detail config   //查看raid卡详细信息

    显示所有物理磁盘信息

    hpssacli ctrl slot=0 pd all show status
hpssacli ctrl slot=0 pd all show detail   //查看所有物理磁盘详细信息
hpssacli ctrl all show config | grep physicaldrive  //查看磁盘总数和磁盘类型

    查看电池状态

    hpssacli ctrl all show status | grep Battery

    查看raid缓存策略

    hpssacli ctrl all show detail config | grep "Drive Write Cache:"
Drive Write Cache: Enabled

// Drive Write Cache 有两种状态：Disable和Enable。
// Disable代表将数据直接写入磁盘，Enable代表先将数据写入缓存然后在写入磁盘。

    常见设置

    开启阵列逻辑盘cache加速策略

    hpssacli ctrl slot=1 logicaldrive 1 modify arrayaccelerator=enable

    没有电池的情况下开启raid写缓存

    hpssacli ctrl slot=1 modify nobatterywritecache=enable forced

    启用物理磁盘cache功能

    hpssacli ctrl slot=1 modify drivewritecache=enable

    设置读写百分比

    hpssacli ctrl slot=1 modify cacheratio=25/75

    raid电池策略

    电池作用

    保护数据：当服务器掉电时，raid电池提供电量保证raid卡缓存的数据写入磁盘。

    电池的充放电

    由于电池是可充电电池，在空闲时候会有微弱的放电。当电量低于一定的阀值，raid卡程序会进行一次放电，将剩余电量放掉然后再充电。在充放电的过程，raid的缓存策略会发生改变，当电池电量低于阀值时，raid卡程序会认为电池不可用，从而禁用了缓存策略，这时数据会直接写入磁盘，raid IO性能会下降，系统的IO的负载会升高。对于高IO的应用，IO性能下降的影响还是比较大。

    电池充放电对IO影响的解决方案

    1.检测电池的状态，掌握电池充放电的周期，在机器负载低的情况下手动强制放电。

    MegaCli -AdpBbuCmd -BbuLearn -aALL

    电池在放电过程策略会从WriteBack变成WriteThrough

    2.强制改变raid策略，使其在充放电过程不禁用raid卡缓存

    MegaCli -LDSetProp CachedBadBBU -lall -a0

    raid缓存策略从No Write Cache if Bad BBU变成Write Cache OK if Bad BBU，这样在充放电的过程IO性能也不下降。

    简单介绍四种raid cache策略

    • WriteBack

    进行写操作时，将数据写入RAID卡缓存，并直接返回，RAID卡控制器将在系统负载低或者Cache满了的情况下把数据写入硬盘。该设置会大大提升RAID卡写性能，绝大多数的情况下会降低系统IO负载。数据的可靠性由RAID卡的BBU(Battery Backup Unit)进行保证。

    • WriteThrough

    数据写操作不使用缓存，数据直接写入磁盘。RAID卡写性能下降，在大多数情况下该设置会造成系统IO负载上升。

    • No Write Cache if Bad BBU

    如果BBU出问题，则关闭Write Cache。由WriteBack自动切换到WriteThrough模式。如果没有特殊要求，强烈建议采用该设置，以确保数据的安全。

    • Write Cache OK if Bad BBU

    如果BBU出问题，依然启用Write Cache. 这是不推荐的设置，BBU出问题将无法保证断电情况下数据的正常，如果此时依然采用WriteBack模式，遇到断电将发生数据丢失。除非有UPS作额外保证，不然不推荐采用这个设置。

  • ipmitool智能平台管理工具

    Sep 15, 2015

    IPMI

    Intelligent Platform Management Interface

    IPMI是一个开放的标准，监控，记录，回收，库存和硬件实现独立于主CPU，BIOS，以及操作系统的控制权。无论在开机还是关机的状态下，只要接通电源就可以实现对服务器的监控。

    The service processor (or Baseboard Management Controller, BMC) is the brain behind platform management and its primary purpose is to handle the autonomous sensor monitoring and event logging features.

    IPMI是一种规范的标准，其中最重要的物理部件就是BMC(Baseboard Management Controller)一种嵌入式管理微控制器，它相当于整个平台管理的“大脑”，通过它ipmi可以监控各个传感器的数据并记录各种事件的日志。

    利用 IPMI可以监视服务器的物理特征，如温度、电压、电扇工作状态、电源供应以及机箱入侵等。

    ipmitool

    ipmitool - utility for controlling IPMI-enabled devices

    ipmitool 用于管理和配置，支持智能平台管理接口(IPMI)1.5版和2.0版规范的设备的命令行工具

    ipmitool常用选项

    Ipmitool本地监控命令：

    ipmitool –I open command

    其中-I open表示使用OpenIPMI接口，command有以下项：

    • raw：发送一个原始的IPMI请求，并且打印回复信息。

    • Lan：配置网络（lan）信道(channel)

    • chassis ：查看底盘的状态和设置电源

    • event：向BMC发送一个已经定义的事件（event），可用于测试配置的SNMP是否成功

    • mc： 查看MC（Management Contollor）状态和各种允许的项

    • sdr：打印传感器仓库中的所有监控项和从传感器读取到的值。

    • Sensor：打印详细的传感器信息。

    • Fru：打印内建的Field Replaceable Unit (FRU)信息

    • Sel： 打印 System Event Log (SEL)

    • Pef： 设置 Platform Event Filtering(PEF)，事件过滤平台用于在监控系统发现有event时候，用PEF中的策略进行事件过滤，然后看是否需要报警。

    • Sol/isol：用于配置通过串口的Lan进行监控

    • User：设置BMC中用户的信息 。

    • Channel：设置Management Controller信道。

    ipmitool 常用命令举例

    查看传感器中的各种监测值和该值的监测阈值，包括CPU温度，电压，风扇转速，电源调制模块温度，电源电压等信息。

    ipmitool -I open sensor list

    查看idrac卡相关信息，包括IP、MAC地址等

    ipmitool -I open lan print

    通过ipmitool重置idrac登陆密码

    1.加载内核模块

    modprobe ipmi_msghandler
modprobe ipmi_devintf
modprobe ipmi_si

    2.查看idrac用户

    ipmitool user list <channel_no>

    DELL的是1，HP的是2

    如果不知道机器的型号可以通过如下命令获取通道号,获取到的通道号代表管理员id

    for i in {1..14}; do ipmitool lan print $i 2>/dev/null | grep -q ^Set && echo $i; done

    3.重置管理员密码

    ipmitool lan set <channel_no> password  xxxxxx
ipmitool lan set 2 password  xxxxxx

  • 再见，青春

    Jun 19, 2015
    那时的你，很年轻

    上海的夏夜凉风袭袭，一个人在出租房里惴惴不安。三天假期的小欢喜无法弥补青葱岁月别离的忧伤。今天发毕业证，而我却在上班。没有亲自去领，一方面是工作的原因，还有一方面可能是自己不想目睹那真切的别离吧~毕业证和毕业照已经邮寄在路上，株洲到上海。像是从故乡寄给在外游子的一封信，里面有你故乡的味道和曾经的记忆。尽管你已经知道一切都已经回不去了，但是你还是很期待那封信，还是想一次又一次的去读它。怀念，不是因为那个时候有多美好，而是因为那个时候你很年轻。

    大学四年，的确没有什么好称道的。平淡之中偶尔精彩。很可惜四年了都没能够认清自己，没有想明白今后的道路要怎么走。既没独善其身，也没随波逐流。过着自己认为理所当然的日子，不去理会那些自己认为毫无价值与意义的人和事。自己总是自以为是，性格上也有些小问题，也不能算是缺陷，反正就是喜欢一个人独自生活，自己安排自己的时间，不想别人打扰，也很少去关心朋友，基本上好友不主动联系我，我是不会打电话去操心他的。反正朋友也不多，也只能怪自己不够坦诚。也说不清缘由吧，不知道是自己太吝啬时间还是不想去打扰别人，空间、朋友圈默默关注及时点赞就好啦。

    四年，给我的感觉并没有四年，回忆里最清晰深刻地只有两个暑假。一个在滇藏线，一个在工大西街。依稀记得当初不明白穷游的真正含义，和一群和我一样不明不白的人在路上定义着穷游。大理的雨夜，丽江的酒吧，虎跳峡的历险，香格里拉的离别与抉择，进藏路上的无助和美景。成为我们永远珍藏的回忆。记得即将离开学校时，和那群虽是不明不白但是历经生死的人还笑谈我们傻逼的曾经。只怪当时太年轻。还好年轻，所以才会有那些美好。工大西街，田径场的夜晚，台球桌上的小憩，闷热机房里的神曲，夜宵摊的喧闹，还有那些年刷过的美团。最努力，最自律，收获最多的那个夏天。或许真的只有涅槃才能重生。一切的付出都是值得的，不会有作废的努力。

    不知如何和青春道别，只知江湖身不由己。

    既自以心为形役，奚惆怅而独悲？悟已往之不谏，知来者之可追。实迷途其未远，觉今是而昨非。

    以前自己怎么也不会想到现在的我会在上海，朝九晚五的工作。那时的我特别想去北京，想亲自体验北漂的生活，也特别佩服那些心怀梦想在北京打拼的人，尽管现实不会如理想中那么美好，但是依然坚持着，因为这里很热闹，你并不会觉得孤单。这里很公平，你并不会缺少机会。只身来到上海，魔都并不如我想象般高贵冷艳，这里有太多的人和我一样，一样年轻，一样憧憬这美好的未来。尽管这里精英荟萃，尽管自己很渺小，我们都坚信：世界是美好的，是值得我们去奋斗的。

    工作其实挺考验一个人的能力，如何把工作八小时效率提高，如何合理的管理安排自己的时间，工作、学习、健身、阅读、娱乐等。工作后尽管有各种限制，并不如你想象般自由有趣。但是依然可以过的很有趣，生活是自己的，你说要有趣就可以有乐趣。

    我只想做一个不安分的人。骑着自行车穿过魔都的每个角落，用相机留住这里的过往，夜晚奔跑在阵阵花香的锦绣大道，或是沉浸在安静的浦图。希望自己的不安分能够给生活带来一些精彩。看着周围的同事，有时会让我很焦虑。房贷、车贷，或许这些我都需要经历吧~不谈也罢。还是喜欢现在一无所有的自己。拥有的越多或许烦恼也就越多。

    最后也不知道想说什么了，只想告诉自己，出了校园入了社会，要做一个成熟理智的男人，学着承担更多的责任。在快节奏和重复的生活里给自己一点惊喜和乐趣。

    要有最朴素的生活和最遥远的梦想，即使明天天寒地冻路远马亡。——海子
